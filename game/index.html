If you have zero experience

Think of a website like a house: HTML is the rooms and walls, CSS is the paint and furniture, JavaScript is the electricity and moving parts.

1

Create the files

Make one folder. Inside it, create index.html and style.css.

2

Write page content

Use headings, paragraphs, links, images, lists, sections, and buttons.

3

Connect the CSS

Add <link rel="stylesheet" href="style.css"> inside the HTML <head>.

4

Style slowly

Start with body, colors, fonts, spacing, then layout. Do not try to style everything at once.

HTML reference

The structure of every page

HTML tells the browser what each part means. Good HTML makes your site easier to style, read, search, and access.

Complete starter HTML

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Website</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <header>
        <nav>Navigation</nav>
    </header>

    <main>
        <section>
            <h1>Main title</h1>
            <p>Short description of the page.</p>
        </section>
    </main>

    <footer>Footer content</footer>
</body>
</html>

Common tags

<h1>
Main title. Use one main H1 per page.
<p>
Normal paragraph text.
<a>
Link to another page, section, file, or website.
<img>
Image. Needs src and helpful alt.
<button>
An action the user can press.

Semantic tags

01

Header

A <header> is usually the top part of a website. It often contains the logo, navigation, and important buttons. If you see a line across the top of a website, that is usually the header area.

  • <header> wraps the top area.
  • <nav> tells the browser this is navigation.
  • <a> creates each clickable link.

Header code

<header class="demo-header">
    <strong>MySite</strong>
    <nav>
        <a href="#">Home</a>
        <a href="#">About</a>
        <a href="#">Contact</a>
    </nav>
</header>

Semantic page structure

Semantic tags describe the job of each page area. They make your code easier to read and help browsers, search engines, and screen readers understand the page.

  • <header> intro or top area.

  • <section> a grouped topic area.

Hero Section

A hero section is the first big area people see. It usually has a headline, short text, and one important button. Use it to explain the purpose of the page fast.

  • <section> groups one topic.
  • <h1> is the main title.
  • <p> explains the title.

Hero code

<section class="hero-box">
    <h1>Learn HTML and CSS</h1>

03

Card

A card is a small box used to group related content. Cards are common in portfolios, product pages, dashboards, blogs, and menus.

  • <article> is good for reusable content blocks.
  • class="card" lets CSS style this box.
  • Padding creates space inside the card.

Card code

<article class="card">
    <h2>Portfolio Project</h2>
    <p>A simple website made with HTML and CSS.</p>
    <button>View project</button>
</article>

04

Button

A button is for an action, like sending a form or opening a menu. A link is for moving somewhere. This difference matters for clean code.

  • Use <button> for actions.
  • Use <a> for navigation.
  • CSS controls the size, color, and hover effect.

Button CSS

.button {
    background: #38bdf8;
    color: #03111f;
    padding: 12px 18px;
    border: 0;
    border-radius: 8px;
}

.button:hover {
    background: #34d399;
}

Result

05

Form

A form collects information. Every input should have a label so people understand what to type, and so screen readers can explain the field.

  • <label> names the field.
  • <input> lets the user type.
  • type="email" asks for an email format.

Form code

<form>
    <label for="email">Email</label>
    <input id="email" type="email">
    <button>Join</button>
</form>

Result

06

Grid Layout

Grid makes rows and columns. The example creates three boxes in one row on big screens. On small screens, CSS can make them stack.

  • display: grid turns on grid layout.
  • grid-template-columns decides columns.
  • gap creates space between boxes.

Grid CSS

.grid {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: 12px;
}

Table

Use a table for real rows and columns of data, like prices, schedules, grades, stats, or comparisons. Do not use tables just to create page layout.

  • <table> wraps the data.
  • <th> is a heading cell.
  • <td> is a normal data cell.

Table code

<table>
    <tr>
        <th>Tag</th>
        <th>Use</th>
    </tr>
    <tr>
        <td>h1</td>
        <td>Main title</td>
    </tr>
</table>

  • <summary> is the clickable title.
  • <details> hides or shows the content.
  • The browser gives it basic interaction for free.

Details code

<details>

Blockquote

Use a blockquote when you show a quote from another person, article, or source. It tells readers and browsers that this text is quoted.

  • <blockquote> wraps the quote.
  • <cite> can name the source.
  • CSS can make it look like a quote card.

Quote code

<blockquote>
    <p>Practice is how code becomes familiar.</p>
    <cite>CodeSheet</cite>

Aside

An aside is content related to the main content, but not the main content itself. Examples: tips, ads, side notes, profile cards, and related links.

  • <aside> means side information.
  • It can appear beside content or inside an article.
  • Great for notes and helpful reminders.

Aside code

<aside>
    <h3>Tip</h3>
    <p>Use semantic tags before styling.</p>

20

Progress

The progress element shows completion. Use it for upload progress, course progress, task progress, or setup steps.

  • value is the current amount.
  • max is the full amount.
  • Browsers render it as a progress bar.

Progress code

<label for="course">Course progress</label>
<progress id="course" value="70" max="100">70%</progress>

Input types

Inputs can ask for different kinds of data. Choosing the right type helps the browser show better controls, especially on phones.

  • type="text" normal typing.
  • type="email" email format.
  • type="date" date picker.

Inputs code

<input type="text" placeholder="Your name">
<input type="email" placeholder="Email">
<input type="date">

Links

A link moves the user somewhere. It can go to another website, another page, or a section on the same page.

  • href is the destination.
  • #section jumps to an id on the page.
  • Links should describe where they go.

Link code

<a href="about.html">Read about us</a>
<a href="#contact">Jump to contact</a>

Lists

Lists are perfect when you have steps, features, ingredients, rules, or a menu. Use unordered lists for normal bullets and ordered lists for numbered steps.

  • <ul> creates bullets.
  • <ol> creates numbers.
  • <li> is one list item.

List code

<ul class="features">

11

Margin vs padding

Padding is space inside the box. Margin is space outside the box. If text feels too close to the edge, use padding. If boxes are too close together, use margin or gap.

Spacing CSS

.box {
    padding: 20px;  /* inside */
    margin: 16px;   /* outside */
}

Layouts

Place things where they belong

Use Flexbox for one direction and Grid for full page/card layouts.

Flexbox

.navbar {
    display: flex;
    align-items: center;
    justify-content: space-between;
    gap: 16px;
}

Best for navigation bars, button rows, centering, and horizontal groups.

Grid

.grid {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
    gap: 20px;
}

Best for galleries, dashboards, product cards, and responsive sections.

Mistakes and fixes

Common beginner problems

Almost every developer hits these. The fix is usually small once you know what to look for.

Problem

CSS is not working

Check the file name, folder location, and the link tag in the HTML head.

<link rel="stylesheet" href="style.css">
Problem

Class style not applying

The HTML class and CSS selector must match exactly.

<div class="card"></div>

.card {
    padding: 20px;
}
Problem

Page has unwanted white edges

Browsers give the body default margin. Remove it in CSS.

body {
    margin: 0;
}
Problem

Cards overflow on mobile

Use flexible columns instead of fixed wide columns.

.grid {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
}

Glossary

Words developers use

A quick A-Z style list for words that appear in tutorials and docs.

Attribute

Extra information inside an HTML tag, like href or class.

Selector

The CSS part that chooses what to style, like .card.

Property

The CSS thing you change, like color or padding.

Value

The setting for a property, like 20px or blue.

Semantic

HTML that describes meaning, like main, nav, and footer.

Responsive

A design that adapts to phone, tablet, and desktop screens.

Viewport

The visible browser area where your website appears.

DOM

The browser's live version of your HTML that JavaScript can change.

Advanced but useful

Level up after the basics

These ideas make websites cleaner, faster to edit, and easier to maintain.

CSS variables

Store colors and spacing in one place so you can change the design quickly.

:root {
    --accent: #38bdf8;
}

a {
    color: var(--accent);
}

Responsive units

Use flexible sizing so your design adapts instead of breaking.

.hero {
    width: min(1100px, 100% - 32px);
    font-size: clamp(2rem, 5vw, 5rem);
}

Accessibility

Use real buttons, labels, alt text, visible focus, and readable contrast.

button:focus-visible {
    outline: 3px solid #38bdf8;
    outline-offset: 3px;
}
